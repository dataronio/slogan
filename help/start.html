The Slogan Programming Language

Introduction

Slogan is a general purpose, dynamically typed, expression-oriented programming language. Slogan makes it easy to write multi-threaded, distributed applications. The current implementation of Slogan comes with an interpreter and an optimizing compiler that generates platform specific binaries.

Installing Slogan

You can fetch the latest source code of Slogan from github. Slogan should build without problems on all modern Unixes - including GNU/Linux and the various BSDs. Once you have cloned the git repository, issue these commands to compile Slogan: 

            $ cd /path-to-slogan-directory
            $ ./configure-platform
            $ make
            $ ./src/slogan
Once you execute the last command, you should see a prompt like this: 
            slogan>
This is the Slogan REPL (Read-Eval-Print-Loop) where you can enter expressions. Slogan will print back the result of evaluating the expressions. Here is are a few expressions that you may want to try: 
            slogan> "hello, world";
            hello, world
            slogan> 1 + 2;
            3
Congratulations for executing your first programs in Slogan! To quit the REPL, type `exit();' and press enter.
To compile Slogan programs from any directory, you have to create the environment variable `SLOGAN_ROOT': 
            $ export SLOGAN_ROOT=/path-to-slogan-directory

Learn Slogan

Slogan is easy to learn. Its syntax must be familiar to most programmers, especially those who have worked with JavaScript or some language in the C family. Slogan's semantics is modeled after the programming language Scheme. This means that you need to master only a small set of concepts to program in the small as well as in the large.

The following tutorial will help you get started programming in Slogan. It assumes no previous programming background but I hope experienced programmers, who are new to functional programming, will also enjoy it. All sample code can be directly entered at the REPL and evaluated. In the text of the tutorial, the result of evaluation will be prefixed by "=>" to differentiate it from program input.
