Distributed Computing

In the previous chapter we saw how a Slogan program can execute mutiple independet tasks simultaneosly. We also saw how these tasks can work together by passing messages to each other. This chapter shows how separate Slogan programs communicate and coordinate their actions by message passing. The programs participating in the distributed computation may run on the same computer or on separate computers connected by a network. Messaging happens over sockets. These are not the same sockets that you might have encountered in networking libraries of other languages. Slogan messaging sockets are objects that can efficiently transfer messages over a number of transports: in-process, inter-process and TCP. Slogan sockets are also highly scalable, taking advantage of event-based IO facilities provided by the underlying operating system. If you are familiar with ZeroMQ you know what this is all about. The messaging framework of Slogan is implemented on top of nanomsg, the next generation ZeroMQ.

This chapter introduces the different patterns in which Slogan programs can form distributed systems.
