Compilation and FFI

This chapter is about using Slogan to write high-performance applications that integrates well with the host operating system.

Compilation

The compile function translates a slogan script into machine code. For example, here is a simple script that we will convert to a system executable:

          // hello.sn
          println ("hello, world!");

This script can be compiled from the REPL as: 

          > compile ("hello", exe = true);

Note:For the compiler to work correctly, make sure you have set the SLOGAN_ROOT environment variable as described in the install instructions.

The exe keyword parameter instructs the compiler to assemble a standalone executable. This executable is created as "hello.run" and can be ran from the command line just like you run any other program.

FFI

The Foreign Function Interface (FFI) is a set of functions that allows a Slogan script to load a shared library (or DLL) and call the functions exported by it. Here is a simple shared library written in C and the Slogan function call that invokes the functions in it:

/* mylib.c */
#include <stdio.h>
#include "slogan.h"
 
void c_hello (void)
{
  printf ("hello from C!\n");
}

int c_add_two_ints (___slogan_obj i, ___slogan_obj j)
{
  return ___int (i) + ___int (j);
}

On a platform with the GCC compiler suite installed, a dynamically loadable library is created using these commands:

          $ gcc -c -I${SLOGAN_ROOT}/platform/gsc/include -I${SLOGAN_ROOT}/src/include -fpic mylib.c 
          $ gcc -shared -o libmylib.so mylib.o

The following sessions shows how to load the library and call the C functions:

          > var mylib = ffi_open ("./libmylib.so");
          > ffi_call_void_0 (mylib, "c_hello");
          >> hello from C!
          > ffi_call_int_2 (mylib, "c_add_two_ints", 10, 20);
          => 30
          > ffi_close (mylib);

Functions for calling into dynamic libraries all start with the prefix ffi_call_***. The next part of the name identifies the return type of the C function and the number of arguments it takes. For example, ffi_call_void_0 calls a C function that take no arguments and returns nothing. ffi_call_int_2 calls a C function that takes two arguments and returns an int. The C functions must accept arguments of type ___slogan_obj which can represent a Slogan object of any type. There are C macros and functions declared in slogan.h for converting a ___slogan_obj to an equivalent C object and vice versa. C functions that takes a single void* as argument can also be called using FFI. The functions ffi_call_with_void_pointer_1 and ffi_call_void_pointer_with_void_pointer_1 are used for this. A C library function to be called using the FFI should not have more that 8 parameters. For passing more values from Slogan to C, we can make use of lists and arrays. A C function can return the following types into Slogan: int, int64, float, double, char* and void*.

See the samples ./example/tutorial/egdlib.c and ./example/tutorial/ffi.sn for more usage scenarios of FFI.
